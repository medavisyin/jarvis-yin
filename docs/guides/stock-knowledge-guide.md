## 1. 股票市场基础

### 什么是股票？为什么要投资股票？

**股票**是公司发行的所有权凭证。你买入一股，就相当于拥有一小块公司权益：公司盈利、成长时，股价往往上涨，你还可能获得分红。可以把公司想象成一家「合伙生意」，股票就是「合伙份额」。

人们投资股票的常见原因包括：分享经济增长与公司成长、获得股息收入、资产配置多样化。**但股价会跌，甚至可能亏光本金**——这是与存款、国债不同的风险等级。

### A股市场简介

**A股**指在中国大陆交易所上市、以人民币计价、主要供境内投资者交易的股票。

| 市场/板块 | 说明 |
|-----------|------|
| **上交所（SSE）** | 大型蓝筹、部分行业龙头集中，传统上大盘股较多 |
| **深交所（SZSE）** | 深市主板、中小板（历史概念，现已并入主板）等 |
| **创业板** | 成长型、创新型企业较多，涨跌幅限制通常更宽 |
| **科创板** | 硬科技、研发投入高的企业，制度与风险特征与主板有差异 |

具体上市规则会调整，投资前以交易所与监管最新规则为准。

### 股票代码规则（记忆口诀）

| 代码特征 | 常见含义 |
|----------|----------|
| **6xxxxx** | 上海主板 |
| **0xxxxx** | 深圳主板等 |
| **3xxxxx** | 创业板 |
| **688xxx** | 科创板 |

### 交易规则（常见情况）

| 项目 | 常见规则 |
|------|----------|
| **T+1** | 当日买入的股票，通常下一个交易日才能卖出 |
| **涨跌停** | 多数普通股票日涨跌幅约 **±10%**；部分板块（如创业板、科创板新股等）可能为 **±20%** 或其他，以实际标的为准 |
| **交易时间** | 上午 **9:30–11:30**，下午 **13:00–15:00**（不含集合竞价等时段细节） |

### 基本术语（OHLCV 的一部分）

| 术语 | 含义 |
|------|------|
| **开盘价（Open）** | 当日第一笔成交价（或集合竞价产生） |
| **收盘价（Close）** | 当日最后一笔成交价，技术分析里最常用 |
| **最高价（High）** | 当日成交的最高价 |
| **最低价（Low）** | 当日成交的最低价 |
| **成交量（Volume）** | 当日成交股数或手数，代表参与度 |
| **换手率** | 成交量与流通股本之比，反映筹码活跃程度 |

💡 **小提示**：Jarvis 的技术分析从本地日线数据 `daily.csv` 读取 OHLCV，再用 **pandas-ta** 计算指标（见 `scripts/stock/technical_analysis.py`）。

## 2. 如何看懂股票行情

### OHLCV 数据解读

一根 **K线（蜡烛图）** 把一天的 Open、High、Low、Close 画在一起：实体表示开盘与收盘之间区间，细线（影线）表示最高、最低。**成交量**常画在K线下方，用来判断「有没有人真金白银参与」。

### K线图基础

- **阳线**：收盘价高于开盘价，通常涂红（A股习惯），表示当天买方相对占优。  
- **阴线**：收盘价低于开盘价，通常涂绿，表示当天卖方相对占优。  
- **上影线**：最高价离实体较远，说明盘中曾冲高但被压回。  
- **下影线**：最低价离实体较远，说明盘中曾下探但被拉回。

可以把它想成「当天多空打架的成绩单」：实体是主战场结果，影线是曾打到多远。

### 常见K线形态（入门）

| 形态 | 大致含义（需结合趋势与位置） |
|------|------------------------------|
| **锤子线** | 下影线长、实体小，常出现在下跌后，可能暗示下方支撑 |
| **十字星** | 开收接近，多空僵持，可能预示变盘 |
| **吞没形态** | 后一根实体完全包住前一根，可能意味趋势反转力量 |
| **早晨之星** | 下跌后出现「阴—小实体—阳」组合，经典底部反转信号之一 |

⚠️ **注意**：单一形态不是圣杯，Jarvis 在技术分析里会识别多种形态并与指标综合参考，仍需你结合大盘与基本面判断。

### 成交量分析

- **放量**：成交量明显高于近期平均，常伴随重要价格突破或情绪集中释放。  
- **缩量**：成交清淡，可能是观望或趋势中继。  
- **量价配合**：涨时放量、跌时缩量，常被视为较健康的多头结构；反之需警惕。

Jarvis 中会计算 **成交量均线**（如 5 日、20 日）并与当日量对比，便于看是否「异常放量」。

## 3. 技术分析入门

### 均线系统（MA）

**均线（Moving Average）** 是过去若干天收盘价的平均值，用来平滑价格波动、看趋势。

| 均线 | 常见理解 |
|------|----------|
| **MA5 / MA10** | 超短期、短期，贴近价格，灵敏但噪音大 |
| **MA20** | 月线级别习惯参考，常作「中期趋势」 |
| **MA60** | 季线习惯参考 |
| **MA120 / MA250** | 半年线、年线，偏长期趋势与牛熊分界参考 |

**用法直觉**：价格在均线上方且均线向上，多头意味强；反之偏空。多条均线「多头排列」常被视为趋势较好。

Jarvis 使用 **pandas-ta** 的 SMA 计算 **MA5、10、20、60、120、250**。

### 金叉与死叉

- **金叉**：短周期均线从下向上穿越长周期均线，常视为多头信号。  
- **死叉**：短周期均线从上向下跌破长周期均线，常视为空头信号。

同样要结合位置：高位死叉与低位金叉意义不同。

### MACD 指标

MACD 由 **DIF（快线）**、**DEA（信号线）** 和 **柱状图（Histogram）** 组成，反映趋势与动量。

- DIF 上穿 DEA → 常称 **MACD 金叉**，动量转强参考。  
- DIF 下穿 DEA → **MACD 死叉**，动量转弱参考。  
- 柱状图扩张/收缩 → 动量加强或减弱。

Jarvis 采用经典参数 **(12, 26, 9)**，与 `pandas-ta` 的 `macd` 一致。

### RSI 指标

**RSI（相对强弱指标）** 衡量一段时间内涨跌力度，范围通常 0–100。

| 区间 | 常见解读 |
|------|----------|
| **> 70** | 超买区，短期可能过热 |
| **< 30** | 超卖区，短期可能过冷 |
| **30–70** | 中性区，需结合趋势 |

Jarvis 使用 **14 日 RSI**（`rsi_14`）。

### KDJ 指标（中国市场常用）

KDJ 由 **K、D、J** 三条线组成，源自随机指标（Stochastic）。A股投资者常看 **J 值**：

- J 过高 → 超买敏感信号  
- J 过低 → 超卖敏感信号  

Jarvis 用 **pandas-ta** 的随机指标 **(9,3,3)**，并据公式 **J = 3K − 2D** 计算 `kdj_j`（见 `technical_analysis.py`）。

### 布林带（Bollinger Bands）

- **上轨 / 中轨 / 下轨**：中轨多为均线，上下轨多为中轨加减若干倍标准差。  
- **收窄（squeeze）**：波动率下降，可能酝酿突破。  
- **价格贴轨**：靠近上轨不一定立刻跌，但常提示短期偏离均值。

Jarvis 使用 **20 日、2 倍标准差** 的布林带，并输出宽度、价格在带内位置等供报告使用。

### 支撑位与阻力位

- **支撑位**：价格下跌时反复止跌的区域，像「地板」。  
- **阻力位**：价格上涨时反复受压的区域，像「天花板」。  

Jarvis 在技术分析结果里会给出 **支撑/阻力** 与近期高低点，便于对照图表。

### ATR（波动率）

**ATR（Average True Range）** 衡量典型波动幅度，不判方向，常用来设止损距离或感知风险。Jarvis 计算 **14 日 ATR**（`atr_14`），报告中可能以占价格的百分比等形式展示。

## 4. 基本面分析入门

### 财务报表「三张表」

| 报表 | 回答的核心问题 |
|------|----------------|
| **利润表** | 赚了多少钱？（收入、成本、费用、利润） |
| **资产负债表** | 家底多厚？（资产、负债、净资产） |
| **现金流量表** | 真金白银进出如何？（经营/投资/筹资现金流） |

### 关键指标详解

| 指标 | 含义 | 入门怎么理解 |
|------|------|----------------|
| **PE（市盈率）** | 股价 / 每股盈利 | 越低通常越「便宜」，但亏损公司 PE 失效；**银行、科技、周期行业**合理区间差很大，要横向比同业 |
| **PB（市净率）** | 股价 / 每股净资产 | 资产重的行业（如银行）更常看 PB；净资产需警惕「账面虚高」 |
| **ROE（净资产收益率）** | 净利润 / 净资产 | 巴菲特等价值投资者极看重长期稳定高 ROE |
| **毛利率 / 净利率** | 毛利、净利占收入比 | 反映定价权、成本控制能力 |
| **负债率** | 负债 / 资产 | 过高可能财务压力大，行业差异大 |
| **营收/利润同比增长** | 与去年同期比 | 反映成长是否在加速或放缓 |

### 估值方法简介

- **PE 估值法**：用「合理 PE × 预期每股盈利」估价格，简单直观，但盈利波动大时易失真。  
- **PEG 估值法**：PE / 盈利增长率，兼顾成长；增长率为负或极不稳定时慎用。

💡 **Jarvis 实现**：基本面数据通过 **akshare** 等拉取并缓存为 `fundamentals.json`，模块 `fundamental_analysis.py` 会按 **盈利、成长、估值、财务健康** 等维度打分（总分 0–100），并生成中文报告。

## 5. 新闻与市场情绪

### 新闻如何影响股价

新闻改变人们对 **未来盈利、风险、政策** 的预期，进而影响买卖力量。同样一条消息，在不同价位、不同大盘环境下市场反应可能完全不同。

### 利好与利空（类型举例）

| 类型 | 示例方向 |
|------|----------|
| **利好** | 业绩超预期、行业政策扶持、重大合同、回购增持 |
| **利空** | 业绩暴雷、监管处罚、大股东减持、行业需求下滑 |

### 情绪分析的作用

情绪分析把大量新闻「压缩」成 **分数或趋势**，帮你快速感知「市场在说什么」，但不能替代你对事实的核实。

### 避免情绪化交易

- 不要因一条推送就全仓进出。  
- 列简单规则：计划买点、卖点、最大亏损比例。  
- Jarvis 的情绪分是辅助工具，**不是买卖指令**。

💡 **Jarvis 实现**：`sentiment.py` 读取个股新闻缓存，通过 **Ollama** 本地 LLM 对每条新闻输出 **−1～+1** 的情绪分与简短理由，再汇总为日报类指标；完整分析流程里会生成 `sentiment_report`。

## 6. 机器学习预测基础（写给新手）

### 什么是机器学习预测？

可以把它想成：**用历史上「当时能知道的信息」去总结规律，再对「现在」给出最可能的分类或数值**。不是算命，是统计意义上的「基于过去相似情况的经验」。

### XGBoost 是什么？

**XGBoost** 是一种 **梯度提升决策树** 集成算法。类比：多个「简单规则专家」依次发言，后面的专家专门纠正前面集体犯的错误，最后投票或加权得到结论。Jarvis 用它做 **未来若干交易日涨跌方向** 的多分类预测（见 `model_xgboost.py`）。

### 特征工程是什么？

原始行情是价格、成交量；模型需要的是整齐的数字列，例如收益率、均线距离、RSI、波动率等。**特征工程**就是把这些加工成模型能吃的「表格」。Jarvis 在 `features.py` 里从 OHLCV + 技术指标构造特征，并定义 **未来 N 日涨跌是否超过阈值** 作为训练目标（默认与计划文档一致：约 **5 日**、**±2%** 分涨/平/跌三档，具体以代码为准）。

### Walk-Forward 验证

**不能用未来数据训练过去**——否则回测会假得离谱。Jarvis 使用 **Walk-Forward**：在一段历史窗口上训练，在紧随其后的短窗口上测试，再向前滑动重复多轮，得到更接近真实的 **样本外** 表现，并输出历史准确率汇总。

### 如何解读预测结果

| 输出项 | 含义 |
|--------|------|
| **prediction** | 模型最可能的类别（涨/平/跌） |
| **confidence** | 最大类别概率，可理解为「模型把握程度」 |
| **probabilities** | 各类别概率分布，可看是否「很纠结」 |
| **feature_importance** | 哪些特征对模型决策贡献大（相关非因果） |
| **walk_forward** | 多轮滑窗测试的命中情况，反映稳健性参考 |

### 预测的局限性

- **过拟合**：在历史数据上很好、实盘失效。  
- **市场非理性**：恐慌与贪婪会让统计规律短期失效。  
- **黑天鹅**：模型无法预测未出现过的极端事件。  

⚠️ **务必把 ML 输出当参考**，结合技术、基本面、风控与你自己的投资计划。

## 7. 如何使用 Jarvis 进行股票分析

### 数据与配置在哪里？

默认数据根目录由环境变量 **`STOCK_REPORTS_ROOT`** 控制（默认 **`C:/reports/stock`**）。个股数据在 `data/{代码}/` 下；关注列表在 **`watchlist.json`**（见 `scripts/stock/config.py`）。

### 添加股票到关注列表

- **API**：`POST /api/stock/watchlist`，JSON 体提供 `symbol`（必填，纯数字代码）、`name`、`sector` 等。  
- **查询列表**：`GET /api/stock/watchlist` 可拉取关注股及最新价概览。  
- **删除**：`DELETE /api/stock/watchlist/<symbol>`。  
- **刷新数据**：`POST /api/stock/watchlist/refresh` 批量更新。

底层由 `watchlist.py` 管理列表与刷新逻辑。

### 运行分析（统一入口）

**`POST /api/stock/analyze`**，JSON 示例字段：

- **`symbol`**：股票代码（纯数字）。  
- **`mode`**：`technical` | `fundamental` | `sentiment` | `xgboost` | **`full`**（默认全流程）。

各模式对应关系简述：

| mode | 得到什么 |
|------|----------|
| `technical` | 技术指标 + 信号 + 技术报告文本 |
| `fundamental` | 拉取/更新财务与估值 + 基本面报告 |
| `sentiment` | 新闻情绪汇总 + 情绪报告 |
| `xgboost` | 训练/预测 + XGBoost 报告 |
| `full` | 以上全部，并额外调用 **`llm_reasoning.generate_prediction`** 生成 **AI 综合预测报告**（`prediction_report`） |

界面侧：Jarvis Web UI 工具栏中有 **「股票分析」** 入口，便于触发上述能力。

### 如何解读综合预测

`llm_reasoning.py` 会把 **技术面摘要、基本面评分与财务要点、新闻情绪、XGBoost 结果（若已有缓存）** 拼成提示，交给 **Ollama** 中配置的较重模型生成连贯中文分析。你阅读时建议：

1. 先看 **事实层**：价格、PE/PB、ROE、情绪分、ML 概率。  
2. 再看 **推理层**：LLM 的论点是否与事实一致、有无夸大。  
3. **交叉验证**：技术多头 + 基本面差 + 情绪极差时，要特别谨慎。

### AI 全市场扫描推荐

除了对单只股票进行深度分析，Jarvis 还提供 **AI 全市场扫描** 功能：

1. **自动扫描**：从全 A 股 ~5000 只股票中自动筛选
2. **三层筛选**：快速过滤 → 技术/情绪分析 → LLM 综合评分
3. **TOP 5 推荐**：给出综合得分最高的 5 只股票，附带推荐理由、风险提示和 **建议买入价位区间**
4. **历史查看**：通过日期下拉框查看历史推荐结果

在 Web UI 中点击 **「🌟 AI推荐」** 按钮即可使用。详细使用说明见《股票使用指南》第 10 节。

**LLM 评分**：现已修复；调用时使用 **`think:false`** 模式（关闭思考链），确保输出正确的 JSON 评分。

⚠️ **注意**：建议买入价位仅为 AI 参考，扫描结果不构成投资建议。请结合自身判断和风险承受能力决策。

### 明日价格预测

在 Web UI 工具栏中点击 **「🎯 价格预测」** 即可进入该功能。系统会针对 **watchlist 中的全部自选股** 分别训练 **XGBoost 回归模型**（与第 6 节里用于涨跌方向分类的用法不同，此处目标是连续价格），并输出对 **下一交易日收盘价、最高价、最低价** 的数值预测。

训练与预测过程中会 **自动追踪历史表现**：例如 **MAPE、MAE** 以及 **方向准确率**（预测涨跌方向是否与实际一致），便于你评估模型是否稳定、是否仅作参考。

**预测校验与健康度**：每次点击**训练**按钮时，系统会自动将**上次预测**与**实际价格**比对，并展示**预测收盘/最高/最低**与**实际收盘/最高/最低**的误差；**模型健康度** 评级（A/B/C/D）用于判断当前预测是否相对可靠。长期使用后若评级**持续为 D**，**建议考虑更换算法**或调整用法。**每次重新训练前**仍会写回已到期预测对应的实际行情，校准统计，避免「只预测、不复盘」。

⚠️ **注意**：价格预测同样属于统计推断，受市场突变与模型局限影响；请与技术面、基本面及风控纪律一并看待，勿单独作为买卖依据。

### 根据分析做投资决策

建议把 Jarvis 当作 **研究助手**：帮你算指标、汇总新闻、给出统计预测。最终 **仓位、止损、是否买入** 必须由你承担后果，并符合你的风险承受能力。

## 8. 投资实战指南

### 新手如何开始

1. 在合规券商 **开户**，搞懂佣金与交易软件。  
2. 先用 **模拟盘** 熟悉下单与情绪管理。  
3. 只用 **闲钱**，绝不借钱炒股。

### 仓位管理

**永远不要全仓一只股票**。常见做法：单票占账户比例设上限（如不超过 10%–20%，因人而异），保留现金应对波动。

### 止损与止盈

- **止损**：提前设定「最多亏多少就认错出场」，可用固定比例或 ATR 倍数。  
- **止盈**：可分档卖出，避免「赚过又吐光」。

### 分散投资

行业、个股、风格适度分散，降低单一暴雷对整体的打击。

### 长线 vs 短线

| 风格 | 特点 |
|------|------|
| **长线** | 更重视基本面与估值，交易频率低 |
| **短线** | 更重视技术与情绪，时间成本高、心理压力大 |

没有高下之分，只有是否适合你。

### 常见新手错误

追涨杀跌、迷信「内幕消息」、过度交易、亏损后加倍赌、把 ML/AI 当圣旨。

### 风险管理原则

不懂不投、不超出承受能力、保留应急资金、定期复盘。

## 8. 全局市场信号（Market Signals）

Jarvis 整合了几个全局指标帮助判断大盘环境：

### 恐慌与贪婪指数（Fear & Greed Index）

CNN 的 Fear & Greed Index 综合衡量市场情绪，范围 0–100。**极度恐惧 (< 25)** 常伴随恐慌性抛售（潜在超跌反弹机会）；**极度贪婪 (> 75)** 信号过热（警惕回调）。Jarvis 通过 `market_sentiment.py` 自动获取并展示。

### VIX 波动率指数

CBOE 波动率指数反映市场对未来 30 天波动的预期。VIX > 30 通常意味着高波动/恐慌状态。

### 黑天鹅事件检测

`black_swan_detector.py` 扫描 Daily Fetch 的国际新闻（`world-news-data.json`），通过关键词匹配（战争、制裁、崩盘、瘟疫等）识别可能的黑天鹅事件，给出严重程度（low / medium / high / critical）和受影响行业。

> 这些指标整合在 UI 的 **"市场信号"** 按钮和 API `/api/stock/signals` 中。
